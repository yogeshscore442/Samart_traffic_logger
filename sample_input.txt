## 3. Full Step-by-Step Guide to Test the Application

### Step 1: Start the Application
1. Open your terminal or command prompt.
2. Navigate to your project directory: `c:\Users\YOGESH\Desktop\new_project_intrn`
3. Make sure your virtual environment is activated (if you use one).
4. Run the application: `python run.py`
5. Open your web browser and go to `http://127.0.0.1:5000`

### Step 2: Authentication (Login)
1. Navigate to the Login page.
2. Enter your credentials.
3. Once logged in, you should be redirected to the administrative Dashboard.

### Step 3: Logging a Violation
1. From the dashboard, click on the "Add Violation" or "New Record" button.
2. Fill in the form using one of the **Sample Inputs** provided above.
3. Submit the form. The system should save the violation to the database and automatically generate a QR code for it.
4. You should see a success message, and the new violation will appear in your dashboard list.

### Step 4: Viewing and Searching Violations
1. On the dashboard, review the table of all logged violations.
2. If you have implemented a search bar, try searching by `Vehicle Number` (e.g., "KA-01").
3. Verify that the correct records are filtered.

### Step 5: Updating Violation Status
1. Find a violation that is currently "Unpaid" in your dashboard.
2. Click on the "Edit", "Update", or "Mark as Paid" button next to it.
3. Change the status from "Unpaid" to "Paid".
4. Save the changes. The dashboard should immediately reflect the updated status.

### Step 6: Public Status Verification (QR Code)
1. On the dashboard, locate the generated QR code or the "View Details" link for a specific violation.
2. If you scan the QR code (or click the public link), it should take you to a public-facing page.
3. This public page should show the violation details and its current status (Paid/Unpaid) without requiring the user to be logged in. This simulates a civilian checking their fine status.
